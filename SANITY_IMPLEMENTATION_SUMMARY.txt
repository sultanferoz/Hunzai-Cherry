WHAT'S BEEN SET UP FOR YOU:
================================================================================

✅ SANITY SCHEMAS (5 Content Types)
   1. Products Schema
      - Product name, emoji, slug
      - Price (number), description (short & full)
      - Category selection (Indoor, Succulent, Flowering, etc.)
      - Images (main + gallery with hotspot)
      - Featured & in-stock status
      - Care instructions array
      - Related video reference
      - Auto publish date

   2. Reviews Schema
      - Customer name, 1-5 star rating
      - Review comment text
      - Avatar image upload
      - Product reference link
      - Verified purchase flag
      - Auto publish date

   3. Hero Section Schema
      - Main headline
      - Subtitle & description
      - CTA button texts (primary + secondary)
      - 3 images (primary, secondary, background)
      - Featured product reference

   4. Videos Schema
      - Title, description
      - Video file upload (MP4, WebM)
      - OR external video URL (YouTube, Vimeo)
      - Thumbnail image
      - Video type classification
      - Related product link
      - Featured flag & duration

   5. Settings Schema
      - Site name & description
      - Logo & favicon
      - Newsletter heading & description
      - Footer text
      - Social media links array

   - getVideos(limit)            → All videos

   - getSettings()               → Global site settings
   - urlFor(image)               → Image URL generator

WHAT YOU NEED TO DO:
================================================================================

1. CREATE SANITY PROJECT (at sanity.io)
   - Sign up/login at https://www.sanity.io
   - Create new project: name="Cherry"
   - Select template: Blank
   - Dataset: production
   - SAVE YOUR PROJECT ID

2. GENERATE API TOKEN
   - Settings > API > Tokens
   - Add new token: name="Cherry Frontend"
   - Permissions: Editor
   - SAVE YOUR API TOKEN

3. ADD TO YOUR APP
   - Create .env.local (copy from .env.example)
   - Add 3 keys:
     NEXT_PUBLIC_SANITY_PROJECT_ID=your-id
     NEXT_PUBLIC_SANITY_DATASET=production
     SANITY_API_TOKEN=your-token

4. INSTALL & RUN
   - pnpm install
   - pnpm dev
   - Visit http://localhost:3000

5. ADD CONTENT
   - Create products in Sanity dashboard
   - Create reviews
   - Upload images & videos
   - Update hero section
   - Publish everything
   - Content appears on your site!

================================================================================

THE 3 KEYS YOU NEED FROM SANITY:
================================================================================

KEY 1: PROJECT ID
   Format: 20-character code (e.g., abc123def456ghijkl)
   Where: sanity.io > Your Project > Settings > API > "Project ID"
   Security: PUBLIC (safe to share)
   Your ID: _________________________________

KEY 2: DATASET
   Standard: production
   Where: sanity.io > Your Project > Settings > API > "Dataset"
   Security: PUBLIC (safe to share)
   Your Dataset: production

KEY 3: API TOKEN
   Format: Long string starting with sk_production_
   Where: sanity.io > Your Project > Settings > API > Tokens > Add
   Security: PRIVATE (keep secret!)
   Permissions: Set to "Editor"
   Your Token: _________________________________

HOW IT WORKS:
================================================================================

   Sanity Dashboard           Your Web App              Your Website
   (Manage Content)      (lib/sanity.client.ts)    (React Components)
        ↓                        ↓                         ↓
   Create Product      →  getProducts()  →  Display on <ProductShowcase />
   Create Review       →  getReviews()   →  Display on <ReviewSection />
   Upload Image        →  urlFor(image)  →  Show optimized image
   Upload Video        →  getVideos()    →  Show video player
   Update Settings     →  getSettings()  →  Update global config
        ↓
   Click "Publish"
        ↓
   Updates appear on your website INSTANTLY!

SECURITY NOTES:
================================================================================

✅ DO:
   - Keep API token in .env.local (not in code)
   - Keep .env.local in .gitignore
   - Set token permissions to "Editor" only
   - Use HTTPS on live site
   - Regenerate token if exposed

❌ DON'T:
   - Commit .env.local to GitHub
   - Share API token
   - Paste token in chat
   - Use token in frontend code
   - Store token in client files

================================================================================

TROUBLESHOOTING:
================================================================================

Problem: "Cannot read properties of undefined"
Solution: Add missing keys to .env.local, restart server

Problem: Content not showing
Solution: Make sure content is "Published" (not draft) in Sanity

Problem: Images not loading
Solution: Check image uploaded in Sanity, set hotspot, refresh

Problem: "Unauthorized" error
Solution: Verify API token permissions, regenerate if needed

Problem: Dev server won't start
Solution: Restart server, check Node.js version (18+), clear .next folder
